Introduction

This API was developed to supplement a developers financial analysis interests. Financial data used to be provided by the major search engines for free through APIs and libraries which were well respected and maintained. After a while, however, these tools were deprecated and could no longer support the needs of financially savvy developers. This API, which relies on screen scraping, provides a method of obtaining information which is already publically available and which used to be provided for free. There is no indication that the information available on Yahoo is protected under copyright and I provide this API with strictly academic and educational intentions. I do not assume liability for the use and/or abuse of this API or its contents.

To read the development documentation for this API, please visit http://austindial.pythonanywhere.com/docs.

Tutorial

Within this section I will describe how to run and interact with the Quote API.

Starting the API

Typically, this API would be run on a local environment through Anaconda Python. However, this version is hosted on PythonAnywhere. By conencting to http://austindial.pythonanywhere.com/, you will access this page with the usage tutorial.

Calling Endpoints

End points are added to the primary URL of the API to instruct the system to perform a given task. In this case, add an endpoint to the following URL http://austindial.pythonanywhere.com/.... For example, to get the quote for Apple Inc. you could call http://austindial.pythonanywhere.com/api/v0.0/get?ticker=AAPL.

.../api/v0.0/create?ticker=TICKER This endpoint will instruct the API to connect to Yahoo Finance and create or update the quote information for the included ticker symbol.

.../api/v0.0/get?ticker=TICKER By calling the get endpoint, the API will return the most recent quote for the given ticker symbol. This is useful for running the API locally when markets are closed.

.../api/v0.0/update?ticker=TICKER If the goal of the API call is to update the database without receiving a quote, this update endpoint can be called. This will update the quote on record for the given ticker without returning the quote to the user. When updating multiple tickers at speed, this endpoint is more efficient than create.

Version History

This API is new, experimental, and is designed mostly to provide a number of key metrics for the development of long term portfolios. The time series, for instance, is not an available from this API yet but could be through a number of extensions of the software. This would allow for a more quantitative analysis of equities.

Beta V0

This is the first beta version of the API. It has a single endpoint with a single argument. This endpoint provides basic stock metrics and quotes given the ticker supplied by the user.

Beta V1 (upcoming)

The next update will hopefully include a second end-point with the ability to call the timeseries of the equity in question. The issue at hand is the sheer volume of information required to pass a time-series through JSON. I will need to research what optimizations can be used to keep the API responsive.
